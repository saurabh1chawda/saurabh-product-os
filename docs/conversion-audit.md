# Conversion Audit

Status: RC1 audit  
Last updated: July 2026

## Executive Summary

The primary conversion is not a purchase or signup. It is recruiter confidence leading to an interview request.

Product OS already supports this through Contact, LinkedIn, Email, and recruiter-oriented pages. The opportunity is to define conversion events clearly and place CTAs after trust moments.

## Primary Conversion Events

| Event | Definition | Priority |
| --- | --- | --- |
| Contact click | Visitor clicks Contact, Email, or LinkedIn from Product OS | P0 |
| Executive Brief completion path | Visitor moves from Executive Brief to Contact or Profile | P0 |
| Product Leadership Brief engagement | Visitor opens JoVE, Logix, Simplilearn, or Mahindra Comviva evidence | P1 |
| GitHub outbound | Visitor opens GitHub profile or repository from Product OS | P1 |
| Profile engagement | Visitor opens Professional Profile from Executive Brief, homepage, or GitHub | P1 |
| Recruiter path selection | Visitor uses Recruiter Tour, For Recruiters, or recruiter reading path | P1 |

## CTA Locations

| Location | Current CTA Role | Conversion Quality | Recommendation |
| --- | --- | --- | --- |
| Homepage hero | Explore product judgment / Professional Profile | Medium | Make Executive Brief primary for recruiters |
| Executive Brief hero | Contact Saurabh / See how I think | Strong | Keep Contact primary |
| Executive Brief Recruiter Toolkit | Profile, LinkedIn, GitHub, Email resources | Strong | Ensure Email and LinkedIn are visually easy to find |
| Professional Profile contact section | Email, LinkedIn, GitHub | Strong | Add "best next step for hiring conversations" copy if not already present |
| Case Studies Hub | Read Product Leadership Brief | Strong for evidence, weaker for conversion | Add Contact CTA after featured brief section |
| Product Leadership Briefs | Read related evidence | Strong evidence path | Add "Discuss this role fit" CTA near bottom |
| GitHub Profile | Product OS / LinkedIn / Email | Strong | Keep Product OS as executive overview |

## Missing Conversion Opportunities

1. Homepage does not make Executive Brief the primary recruiter conversion path.
2. Product Leadership Brief pages can end in more exploration instead of interview request.
3. GitHub repository READMEs should consistently link back to Executive Brief and Contact once public.
4. LinkedIn Featured assets should include Contact or Executive Brief as a fast conversion path.
5. Professional Profile should clearly state that tailored resumes are handled through applications and this page provides Product OS context.

## Recommended Conversion Funnel

```text
Awareness: LinkedIn / GitHub / Search
  |
Orientation: Executive Brief
  |
Evidence: Product Leadership Brief or Professional Profile
  |
Validation: GitHub artifact or Operating Principles
  |
Conversion: Contact / LinkedIn / Email
```

| Metric | Why It Matters |
| --- | --- |

| Contact CTA clicks | Measures interview intent |
| LinkedIn clicks | Measures recruiter follow-up |
| GitHub outbound clicks | Measures artifact credibility interest |
| Product Leadership Brief clicks | Measures evidence engagement |
| Profile views | Measures career-fit evaluation |

## Highest-Impact Experiments

1. Homepage primary CTA: Executive Brief vs Decision OS.
2. Case Studies Hub featured CTA: "Read Product Leadership Brief" plus "Contact after review."
3. Executive Brief final CTA: Contact vs Professional Profile.
4. GitHub profile "Start Here": Executive Brief vs Product OS homepage.

## Validation

No redesign required. Recommendations focus on CTA clarity, event definition, and placement after trust signals.
