# Repurposing Framework

Status: RC1 framework  
Last updated: July 2026

## Executive Summary

The repurposing framework explains how one Product OS artifact becomes multiple publishing outputs without duplicating content.

The system starts with a canonical artifact, extracts one reusable idea, adapts it by channel, and links every output back to the source.

## Core Model

```text
Artifact
  |
Single lesson
  |
Audience angle
  |
Channel format
  |
Source CTA
```

## Repurposing Rules

1. Start from a finished artifact.
2. Extract one idea, not the whole asset.
3. Choose one audience.
4. Adapt format to channel behavior.
5. Link back to the canonical source.
6. Measure qualified interest.

## Artifact To Output Pattern

| Source Artifact | Extract | Output Examples |
| --- | --- | --- |
| Product Leadership Brief | Decision principle | LinkedIn post, blog section, speaking story, interview prompt |
| Framework | Decision method | GitHub README section, workshop worksheet, LinkedIn carousel outline |
| Toolkit | Reusable template | GitHub download, workshop exercise, newsletter checklist |
| Executive Brief | Recruiter summary | LinkedIn profile link, hiring email reference, GitHub profile CTA |
| Worked example | Applied lesson | Blog walkthrough, workshop case, interview discussion prompt |

## Channel Adaptation Guide

| Channel | Format | Recommended Length | CTA |
| --- | --- | ---: | --- |
| LinkedIn | Insight, lesson, short framework | 150-300 words | Source artifact |
| GitHub | README, template, example | As long as needed | Product OS context |
| Blog | Explanation with examples | 800-1,500 words | Related Product OS asset |
| Newsletter | Curated lesson | 500-900 words | One next read |
| Workshop | Agenda and exercise | 30-90 minutes | Toolkit or template |
| Speaking | Narrative and principle | 10-30 minutes | Executive Brief or case hub |

## Editorial Checklist

Before repurposing, answer:

- What is the canonical source?
- What is the single lesson?
- Who is the audience?
- What should the visitor do next?
- What metric will show success?

## Anti-Patterns

- Turning every artifact into a long post.
- Publishing the same explanation in multiple places.
- Creating new metrics that do not exist in the source.
- Posting framework theory without a practical next step.
- Treating LinkedIn engagement as the only success measure.

## Operating Principle

Repurpose the insight, not the entire artifact.
