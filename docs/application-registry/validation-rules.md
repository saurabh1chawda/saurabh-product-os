# Validation Rules

Last updated: 2026-07-18

Validation covers:

- schema compliance
- unique IDs
- valid references
- lifecycle transitions
- event chronology
- application/event consistency
- task consistency
- contact references
- date validity
- terminal-stage consistency
- outcome consistency
- archive consistency
- Git-ignore coverage
- private data segregation
- Resume OS traceability
- duplicate detection
- index consistency

Any private-data commit risk is P0.
