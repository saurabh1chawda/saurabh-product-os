# Content Library

Status: RC1 asset inventory  
Last updated: July 2026

## Executive Summary

The content library maps canonical Product Leadership Platform assets to publishing outputs.

The purpose is to make publishing repeatable without duplicating Product OS. Each asset should produce multiple channel-specific outputs while linking back to its source.

## Content Reuse Rules

- Never copy full Product OS sections into a channel post.
- Extract one idea per output.
- Link to the canonical asset.
- Keep metrics consistent with the source.
- Use GitHub for reusable assets and Product OS for executive narrative.

## Operating Principle

One artifact should create many entry points, but only one source of truth.
