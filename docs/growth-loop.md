# Growth Loop

Status: RC1 acquisition loop  
Last updated: July 2026

## Executive Summary

The growth loop connects Product OS, GitHub, publishing, interviews, and new product artifacts into a continuous acquisition and learning system.

The loop is designed to create sustainable qualified discovery without fragmenting the platform's source of truth.

## Growth Loop Model

```text
Product OS source artifact
  |
Publishing output
  |
Qualified audience discovery
  |
Evidence engagement
  |
Conversation or artifact reuse
  |
Learning signal
  |
Product OS or GitHub improvement
  |
New source artifact
```

## Loop Stages

### 1. Product OS Source Artifact

Product OS or a flagship repository provides the canonical asset.

Examples:

- Executive Brief.
- Product Leadership Brief.
- AI Product Playbook.
- Product Discovery Toolkit.
- AI Prioritization Framework.
- Decision Memo Template.

### 2. Publishing Output

The production system turns the asset into a channel-native output.

Examples:

- LinkedIn insight.
- GitHub release notes.
- Newsletter section.
- Blog post.
- Workshop description.
- Speaking abstract.

### 3. Qualified Audience Discovery

The output reaches a specific audience.

Audiences:

- Recruiters.
- Hiring managers.
- Product managers.
- AI Product Managers.
- Engineering leaders.
- Founders.

### 4. Evidence Engagement

The audience clicks into the canonical source.

Examples:

- Executive Brief view.
- Product Leadership Brief view.
- GitHub repository visit.
- AI Product Playbook engagement.
- Contact page visit.

### 5. Conversation Or Artifact Reuse

The visitor takes a meaningful action.

Examples:

- Recruiter outreach.
- Interview request.
- Repository star or fork.
- Template reuse.
- Workshop inquiry.
- Hiring manager follow-up.

### 6. Learning Signal

The interaction reveals what creates trust.

Examples:

- Recruiters consistently ask for one role-fit brief.
- Hiring managers mention a specific product principle.
- GitHub visitors reuse a template.
- AI PMs engage with prioritization and trust content.

### 7. Platform Improvement

Use learning signals to improve Product OS or GitHub.

Examples:

- Improve CTA copy.
- Add a clearer cross-link.
- Strengthen a README.
- Add a worked example.
- Update a Product Leadership Brief.

### 8. New Source Artifact

The improved artifact becomes the next source for publishing.

## Loop Metrics

| Stage | Metric |
| --- | --- |
| Source artifact | Asset readiness and cross-link completeness |
| Publishing output | Channel clicks and qualified engagement |
| Discovery | Qualified visitors by channel |
| Evidence engagement | Executive Brief, PLB, Playbook, GitHub visits |
| Conversion | Contact clicks, DMs, interview requests |
| Learning | Qualitative mentions and repeated questions |
| Improvement | Updated source artifacts and new release notes |

## Feedback Rules

- Do not change positioning based on one weak signal.
- Prioritize repeated signals from qualified audiences.
- Update the canonical source before publishing new derivative content.
- Treat questions from recruiters and hiring managers as conversion insight.
- Treat GitHub reuse as artifact value insight.

## Operating Principle

The platform grows when every qualified interaction makes the next artifact clearer.
