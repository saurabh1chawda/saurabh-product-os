# Launch Playbooks

Status: RC1 launch system  
Last updated: July 2026

## Executive Summary

Launch playbooks define repeatable acquisition motions for repository launches, framework launches, Product Leadership Brief launches, and Product OS updates.

These are not campaign plans. They are reusable launch operating models that route attention back to canonical Product OS and GitHub assets.

## Launch Rules

- Launch only when the canonical source is ready.
- Use one primary audience.
- Use one primary CTA.
- Link back to Product OS or the flagship repository.
- Measure qualified actions, not only impressions.

## Repository Launch Playbook

Use for:

- AI Product Playbook repository.
- Product Discovery Toolkit.
- AI Prioritization Framework.
- Decision Memo Template.
- Product Leadership Briefs repository.

Pre-launch:

- README complete.
- Status clear.
- Product OS links included.
- Example or template available.
- Metadata and license documented.

Launch assets:

- GitHub release notes.
- LinkedIn announcement.
- Product OS cross-link update.
- Newsletter mention.

Distribution path:

```text
LinkedIn / GitHub
  |
Repository
  |
Product OS context
  |
Related brief or contact path
```

Success metrics:

- Repository visits.
- Stars.
- Product OS return clicks.
- Artifact reuse.

## Framework Launch Playbook

Use for:

- AI Prioritization Framework.
- AI Product lifecycle.
- Decision Writing Pyramid.
- Evidence Ladder.

Pre-launch:

- Framework has a clear problem.
- Inputs and outputs are defined.
- Example exists.
- CTA points to source artifact.

Launch assets:

- LinkedIn insight.
- LinkedIn carousel outline.
- Blog or newsletter explanation.
- GitHub framework page.

Distribution path:

```text
Framework insight
  |
Framework source
  |
Worked example
  |
Related Product OS evidence
```

Success metrics:

- Framework page visits.
- Repository visits.
- Content reuse rate.
- Workshop interest.

## Product Leadership Brief Launch Playbook

Use for:

- JoVE.
- Logix.
- Simplilearn.
- Mahindra Comviva.

Pre-launch:

- Brief complete.
- Metrics validated.
- Role-fit label defined.
- Related resources linked.
- Contact path present.

Launch assets:

- Decision lesson post.
- Case study summary.
- Newsletter feature.
- Interview-prep reference.

Distribution path:

```text
Decision lesson
  |
Product Leadership Brief
  |
Operating Principles
  |
Contact
```

Success metrics:

- Brief views.
- Brief to Contact clicks.
- Hiring manager engagement.
- Interview references.

## Product OS Update Playbook

Use for:

- Executive Brief updates.
- Navigation improvements.
- Recruiter experience changes.
- New flagship surface launches.

Pre-launch:

- Page validated.
- CTA path clear.
- Metadata updated if needed.
- No dead ends.

Launch assets:

- LinkedIn update.
- GitHub profile README update if relevant.
- Newsletter note.
- Recruiter outreach reference.

Distribution path:

```text
Product OS update
  |
Executive Brief
  |
Role-fit evidence
  |
Contact
```

Success metrics:

- Executive Brief views.
- Contact conversions.
- Path completion.
- Returning visitors.

## Launch Retrospective

After every launch, capture:

- What was launched.
- Primary audience.
- Top channel.
- Conversion path.
- Best signal.
- Friction observed.
- Next improvement.

## Operating Principle

Launches should increase trust in the platform, not just announce that something exists.
