# Editorial Calendar Framework

Status: RC1 calendar system  
Last updated: July 2026

## Executive Summary

This document defines a repeatable editorial calendar framework for the Product Leadership Platform.

It is not a fixed content calendar. It is a planning system for deciding what to publish, when to publish it, and how each output connects back to Product OS and flagship repositories.

## Calendar Principles

- Publish from source artifacts.
- Rotate across audiences.
- Balance credibility, usefulness, and discoverability.
- Prioritize quality over volume.
- Use launch moments when artifacts become more useful.

## Weekly Cadence

Recommended weekly rhythm:

| Week Activity | Purpose |
| --- | --- |
| Monday | Select source artifact and audience |
| Tuesday | Draft one insight or framework output |
| Wednesday | Review source accuracy and CTA |
| Thursday | Publish or schedule |
| Friday | Review signals and note follow-up |

Minimum viable cadence:

- One LinkedIn insight per week.
- One GitHub or Product OS link included when relevant.
- One monthly deeper asset.

## Monthly Theme Model

Each month should focus on one dominant theme while still allowing supporting posts.

| Theme | Source Assets | Best Channels |
| --- | --- | --- |
| AI product judgment | AI Product Playbook, AI Prioritization Framework | LinkedIn, GitHub, workshop |
| Discovery and evidence | Product Discovery Toolkit, JoVE Brief | LinkedIn, blog, workshop |
| Product leadership decisions | Product Leadership Briefs, Operating Principles | LinkedIn, newsletter, speaking |
| Executive communication | Decision Memo Template | GitHub, blog, workshop |
| Recruiter confidence | Executive Brief, Professional Profile | LinkedIn, GitHub profile, interview prep |

## Product Launch Moments

Use launch moments when a source artifact becomes meaningfully stronger.

Examples:

- Product OS release.
- Executive Brief refresh.
- New Product Leadership Brief.
- Flagship repository reaching GA readiness.
- New worked example.
- New visual diagram or downloadable template.

Launch moment output set:

| Output | Purpose |
| --- | --- |
| LinkedIn announcement | Awareness |
| GitHub README update | Artifact usability |
| Product OS link update | Source navigation |
| Newsletter note | Deeper audience context |
| Interview prep reference | Recruiter conversion |

## Framework Releases

Use framework releases when a reusable model becomes independently useful.

Release checklist:

- Canonical source exists.
- README or Product OS page explains the framework.
- Example or template exists.
- CTA points to the source.
- Related Product OS resources are linked.

Suggested release sequence:

1. AI Prioritization Framework.
2. Product Discovery Toolkit.
3. Decision Memo Template.
4. AI Product Playbook lifecycle.

## Suggested Monthly Operating Cycle

| Week | Primary Output | Secondary Output |
| --- | --- | --- |
| Week 1 | Product Leadership Brief insight | Executive Brief CTA |
| Week 2 | Framework or toolkit lesson | GitHub repository link |
| Week 3 | Practical template or checklist | Newsletter section |
| Week 4 | Synthesis post or launch update | Metrics review |

## Editorial Backlog Fields

Maintain a lightweight backlog with:

- Source artifact.
- Category.
- Audience.
- Working thesis.
- Channel.
- CTA.
- Status.
- Publish date.
- Metric to watch.

## Governance

- Do not publish a launch post for an unfinished artifact.
- Do not create weekly posts that lack a source link.
- Do not rotate themes so quickly that the audience cannot understand the platform.
- Review monthly performance against recruiter and qualified opportunity metrics.

## Operating Principle

The calendar should create momentum without forcing content that the platform cannot support.
