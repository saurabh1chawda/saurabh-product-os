# GitHub Metadata Standard

Status: RC1 launch standard  
Last updated: July 2026

## Purpose

This document defines recommended GitHub metadata for the six flagship products in the Product Leadership Platform.

Metadata should make the ecosystem discoverable, professional, and easy to understand without creating duplicate positioning.

## Portfolio-Wide Defaults

| Setting | Standard |
| --- | --- |
| Visibility | Public at GA after license and content-boundary review |
| Default branch | `main` |
| Issues | Enable after contribution model is ready |
| Discussions | Optional; enable only if community learning becomes a goal |
| Wiki | Disabled; use versioned docs in the repository |
| Projects | Optional for public roadmap tracking |
| Social preview | Use a clean 1280 x 640 or 1200 x 630 product-specific preview image |

## Topic Taxonomy

Use a small shared taxonomy plus product-specific topics.

### Core Topics

- `product-management`
- `product-leadership`
- `product-strategy`
- `product-frameworks`
- `product-operations`

### AI Topics

- `ai-product-management`
- `ai-strategy`
- `ai-product-frameworks`

### Product Category Topics

- `product-discovery`
- `product-prioritization`
- `case-study`
- `decision-making`
- `templates`

### Technical Topics

Use only where relevant:

- `nextjs`
- `typescript`
- `tailwindcss`
- `vercel`

Avoid duplicate or overly broad tags such as `ai`, `product`, `pm`, or `portfolio` unless they materially improve discoverability.

## Metadata QA Checklist

- Repository description is under 160 characters where practical.
- Homepage points to the most relevant Product OS page.
- Topics use the shared taxonomy.
- Social preview reflects the product, not generic branding.
- Default branch is `main`.
- README status matches repository maturity.
- License and reuse boundaries are visible before public promotion.
