# Cross-Link Strategy

Status: RC1 navigation model  
Last updated: July 2026

## Executive Summary

The cross-link strategy ensures every channel and Product OS page guides visitors toward the next highest-value resource.

No visitor should reach a dead end. Every page, repository, and publishing output should offer a clear next step based on audience intent.

## Cross-Link Principles

1. Product OS is canonical.
2. Executive Brief is the fastest recruiter entry point.
3. Product Leadership Briefs provide role-fit evidence.
4. GitHub provides reusable artifacts.
5. Contact is the conversion path for hiring interest.
6. Every content output should have one primary CTA.

## No-Dead-End Requirements

Every Product OS page should include at least one of:

- Continue Exploring.
- Contact.
- Related Product OS.
- Related GitHub resource.
- Executive Brief.

Every GitHub README should include:

- What this repository is.
- How to use it.
- Related Product OS link.
- Status.
- Next resource.

Every publishing output should include:

- One source link.
- One CTA.

## Link Review Cadence

- Weekly during launch.
- Monthly after baseline stabilizes.
- Immediately after adding or updating a flagship artifact.

Review for:

- Broken links.
- Outdated coming-soon language.
- Missing Contact paths.
- Generic CTA labels.
- Pages without a next step.

## Operating Principle

Every link should reduce uncertainty about what to read next.
