# Editorial Checklist

Status: RC1 checklist  
Last updated: July 2026

## Executive Summary

The editorial checklist verifies that every content draft is accurate, useful, source-linked, and ready for publishing handoff.

Use this checklist before any LinkedIn post, blog, newsletter, GitHub release, workshop description, speaking abstract, or interview story moves into publishing.

## Checklist Summary

| Area | Pass Criteria |
| --- | --- |
| Accuracy | Claims match the source artifact |
| Canonical source | Product OS or GitHub source is linked |
| One key idea | Draft does not try to explain everything |
| Audience fit | The output answers a real audience question |
| CTA | One next action is clear |
| Unsupported claims | No invented metrics, companies, outcomes, or examples |
| Reusability | The output teaches something useful beyond promotion |

## 1. Accuracy

Verify:

- Metrics match the source.
- Company names match the source.
- Product names match the source.
- Role and timeline details match the source.
- The conclusion follows from the evidence.

Decision:

- Pass.
- Needs source correction.
- Reject.

## 2. Canonical Source Linked

Verify:

- The source artifact exists.
- The CTA points to the correct source.
- Product OS remains the narrative source.
- GitHub remains the reusable artifact source.

Required:

- At least one canonical source link in every publishable asset.

## 3. One Key Idea

Verify:

- The draft can be summarized in one sentence.
- The opening and CTA support the same idea.
- Secondary points do not distract from the main lesson.

If the draft has multiple ideas, split it into multiple future backlog items.

## 4. Audience Fit

Verify:

- The audience is explicitly named in metadata.
- The content answers a question that audience cares about.
- The examples and CTA match the audience.

Audience examples:

- Recruiter.
- Hiring manager.
- Product leader.
- AI Product Manager.
- Engineering leader.
- Product practitioner.

## 5. CTA

Verify:

- There is one primary CTA.
- The CTA is specific.
- The CTA points to a source or conversion path.

Preferred CTAs:

- Read the Executive Brief.
- Review the Product Leadership Brief.
- Explore the framework.
- Use the template.
- Contact Saurabh.

## 6. No Unsupported Claims

Reject drafts that include:

- Invented metrics.
- Unverified customer details.
- New company claims.
- Claims that exaggerate AI maturity.
- Generic claims not supported by Product OS.
- Confidential details not already public.

## 7. Reusability

Verify:

- The content leaves the reader with a useful lesson.
- The output is not only promotional.
- The artifact points to a reusable framework, decision, template, or principle.

## Final Review Status

Use one status:

- Ready for publishing handoff.
- Needs editorial revision.
- Needs evidence correction.
- Needs source update.
- Reject.

## Publishing Handoff Requirements

Before handoff, include:

- Final draft.
- Metadata.
- Source link.
- CTA.
- KPI.
- Review status.
- Owner.

## Operating Principle

If the claim cannot be traced to a source, it does not publish.
