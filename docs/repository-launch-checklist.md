# Repository Launch Checklist

Status: Reusable flagship checklist  
Last updated: July 2026

This checklist defines the launch standard for Product OS and future flagship repositories in the GitHub Product Leadership Hub.

## Core Readiness

- [x] README complete
- [ ] License added
- [x] Changelog present
- [ ] Repository topics configured
- [ ] Homepage URL configured
- [ ] Screenshots added
- [ ] Status badge added
- [x] Last updated date present
- [x] Local setup documented
- [ ] Local setup verified immediately before public launch
- [ ] Links validated
- [x] Documentation reviewed

## README Readiness

- [x] Executive Summary
- [x] Problem Statement
- [x] Target Audience
- [x] Key Capabilities
- [x] Screenshots section
- [x] Repository Structure
- [x] Local Development
- [x] High-Level Architecture
- [x] Product OS <-> GitHub relationship
- [x] Roadmap
- [x] Contributing
- [x] License
- [x] Status
- [x] Last Updated

## Documentation Readiness

- [x] Architecture document
- [x] Repository metadata checklist
- [x] Repository launch checklist
- [x] Governance document
- [x] GitHub Product Leadership Hub strategy
- [x] Repository audit
- [x] Flagship product specifications
- [ ] Public contribution guide
- [ ] Issue templates
- [ ] Pull request template

## Product Readiness

- [x] Repository reflects the Product OS product specification
- [x] Audience and problem are clear
- [x] Value proposition is clear
- [x] Roadmap is documented
- [x] Relationship to Product OS and GitHub Hub is clear
- [ ] Screenshots show the strongest product surfaces
- [ ] Public license decision is complete

## Validation Before Public Launch

- [ ] `pnpm lint`
- [ ] `pnpm typecheck`
- [ ] `pnpm build`
- [ ] Production URL checked
- [ ] Open Graph preview checked
- [ ] Key recruiter paths checked
- [ ] Key GitHub links checked
- [ ] Contact links checked

## Future Flagship Repository Standard

Every future flagship repository should complete this checklist before being pinned or promoted from Product OS.
