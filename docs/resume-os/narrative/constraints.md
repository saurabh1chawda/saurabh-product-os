# Resume OS Narrative Constraints

## Protected Fields

The Narrative Layer must not change:

- Candidate identity.
- Employment dates.
- Company names.
- Canonical titles.
- Achievement IDs.
- Bullet IDs.
- Metrics.
- Product OS URLs.
- GitHub URLs.
- Project classification.
- Simulation labels.
- Draft status.

## Allowed Transformations

- Shorten.
- Split long sentences.
- Merge repetitive clauses.
- Reorder clauses.
- Improve rhythm.
- Improve transitions.
- Remove filler.
- Reduce redundancy.
- Improve executive tone.
- Vary sentence openings.
- Strengthen verbs when meaning is unchanged.
- Reduce passive voice.
- Remove generic leadership language.
- Improve readability.

## Disallowed Transformations

- Adding evidence.
- Changing evidence.
- Exaggeration.
- Inferred ownership.
- Stronger metrics.
- Stronger claims.
- New technology claims.
- New domain claims.
- New leadership scope.

## Human Review Gate

Narrative output is never final. It remains `human_review_required` until reviewed and exported.
