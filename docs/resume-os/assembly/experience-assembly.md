# Experience Assembly

## Role Block Fields

For every role:

- Canonical title.
- Company.
- Dates.
- Location when available.
- Optional one-line scope statement.
- Selected bullets.
- Evidence IDs.
- Competency coverage.
- Word count.
- Role relevance score.

## Rules

- Preserve chronology.
- Preserve exact dates.
- Preserve canonical titles.
- Prefer bullets from the primary archetype library.
- Use secondary-archetype bullets only for distinct evidence.
- Avoid multiple variants of the same achievement.
- Avoid excessive evidence concentration from one company.
- Current role must be represented.
- Strong recent roles receive more space.
- Do not force every major achievement into every resume.
- Balance business, product, technical, and leadership evidence.
- Do not overuse led, owned, or drove.
- Every bullet must be interview-defensible.

## Default Company-Level Bullet Limits

| Role | Default |
| --- | --- |
| Current role | 4 bullets; 5 only when strongly justified |
| JoVE | 3 bullets; 4 for enterprise SaaS or discovery-heavy roles |
| Simplilearn | 3 bullets; 4 for growth or monetization roles |
| Freelancer | 1 bullet; 2 for consulting or customer strategy roles |
| Mahindra Comviva | 2 bullets; 3 for payments/platform roles |
| Data Analyst | 0-1 bullet; include for AI, analytics, or data-heavy roles |
