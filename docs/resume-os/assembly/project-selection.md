# Project and Product OS Selection

## Options

A. Company-specific project.  
B. Product OS proof module.  
C. Product Leadership Brief.  
D. GitHub project.  
E. None.

## Company-Specific Project

Use when:

- It is highly relevant.
- It demonstrates missing but honest adjacent capability.
- It is clearly labeled simulation, prototype, or case study.
- It strengthens rather than distracts.

## Product OS

Use when:

- It demonstrates repeatable product judgment.
- It provides relevant public evidence.
- The role values written communication or frameworks.
- No stronger company-specific project exists.

## Use Both Only When

- The role is highly strategic or technical.
- The resume remains two pages.
- They provide different evidence.
- Human review explicitly approves.

## Never

- Present LoyaltyIQ or another simulation as employer work.
- Use simulated metrics as real outcomes.
- Allow project detail to overshadow production experience.
