# Section Budgeting

## Initial Budgets

| Section | Budget |
| --- | --- |
| Header | 2-3 lines |
| Summary | 70-110 words; maximum 5 lines where possible |
| Current role | 4-5 bullets; 18-32 words per bullet |
| Previous two major roles | 3-4 bullets each |
| Older roles | 1-2 bullets each |
| Company-specific project | 3-5 bullets only when strategically valuable |
| Skills | 2-4 concise categories |
| Tools | One compact line or remove |
| Certifications | One compact line |
| Education | 2 concise lines |

## Trimming Order

1. Remove redundant tools.
2. Remove low-priority certifications.
3. Compress older roles.
4. Remove weaker bullets.
5. Remove repetitive leadership language.
6. Reduce project depth.
7. Remove optional Product OS summary sentence.
8. Never remove high-relevance verified evidence merely to preserve decorative sections.

## Warnings

- Summary over 110 words.
- Bullet over 35 words.
- More than five bullets in one role.
- More than 18 total experience bullets.
- More than 20 skills.
- More than one large project section.
- More than five links.
- Estimated page count above two.
