# Assembly Safety Rules

## P0 Failures

- Unsupported claim.
- Modified metric.
- Modified date.
- Modified title.
- Simulation presented as production.
- Broken achievement reference.
- Orphan bullet.
- Missing canonical source.
- Confidential content exposure.
- Broken required URL.
- Work authorization invented.
- Relocation preference invented.

## P1 Warnings

- Evidence concentration.
- Weak domain match.
- Excessive management emphasis for IC role.
- Excessive technical emphasis for growth role.
- Missing recent evidence.
- Resume over two pages.
- Summary too generic.
- Product OS overemphasis.
- Company-specific project too prominent.
- Keyword coverage below target.

## P2 Warnings

- Minor wording repetition.
- Optional certification omission.
- Tool-list cleanup.
- Formatting consistency.
- Low-value older evidence.

## Enforcement

P0 failures stop generation. P1 warnings require explicit human review. P2 warnings are handled during polish.
