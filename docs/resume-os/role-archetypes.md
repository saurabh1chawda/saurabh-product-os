# Role Archetypes

## Executive Summary

Resume OS classifies each job into one primary role archetype and up to two secondary archetypes. This prevents generic tailoring and helps select the strongest headline, summary, bullets, skills, projects, and Product OS proof.

One job may combine multiple archetypes. For example, an AI platform growth role could map to:

- Primary: AI Product Manager
- Secondary: Platform Product Manager
- Secondary: Growth Product Manager

## Archetype Selection Rule

| Step | Decision |
| --- | --- |
| Primary archetype | The role's main hiring problem |
| Secondary archetype 1 | The strongest adjacent competency |
| Secondary archetype 2 | A meaningful differentiator, only if supported |

Do not select more than two secondary archetypes. Too many archetypes dilute the resume.

## AI Product Manager

| Dimension | Guidance |
| --- | --- |
| Core hiring problem | Can this person identify useful AI opportunities and turn them into reliable products? |
| Expected competencies | AI opportunity assessment, prioritization, trust, measurement, workflow design |
| Typical metrics | Adoption, task completion, quality, risk reduction, cost-to-serve, AI feature usage |
| Technical signals | LLM concepts, RAG vs agent reasoning, data readiness, evaluation, human-in-the-loop |
| Leadership signals | Cross-functional AI decision-making, expectation management, responsible rollout |
| Product OS assets | AI Product Playbook, AI Prioritization Framework, Trust Before Automation |
| Best-fit evidence | Logix, AI Product Studio, AI decision systems |
| Skills to prioritize | AI strategy, prompt/evaluation thinking, trust, metrics, platform readiness |
| Risks to avoid | Fabricated AI deployments, vague AI enthusiasm |
| Interview themes | AI opportunity quality, evaluation, trust, build vs buy, lifecycle decisions |

## Generative AI Product Manager

| Dimension | Guidance |
| --- | --- |
| Core hiring problem | Can this person turn generative AI capabilities into usable workflows? |
| Expected competencies | Workflow analysis, prompt strategy, model limitations, safety, UX, measurement |
| Typical metrics | Output acceptance, time saved, quality scores, user trust, workflow completion |
| Technical signals | LLM behavior, retrieval, agents, guardrails, evaluation |
| Leadership signals | Alignment between product, engineering, data, design, and risk |
| Product OS assets | Workflow-to-Agent, RAG vs Agent, AI Success Metrics |
| Best-fit evidence | Logix, AI Product Playbook, GitHub AI frameworks |
| Skills to prioritize | Generative AI workflows, experimentation, model evaluation, trust |
| Risks to avoid | Claiming model expertise beyond product scope |
| Interview themes | Where AI should not be used, workflow fit, evaluation quality |

## Platform Product Manager

| Dimension | Guidance |
| --- | --- |
| Core hiring problem | Can this person sequence platform investments that unlock business outcomes? |
| Expected competencies | Platform strategy, API thinking, scalability, roadmap sequencing, stakeholder alignment |
| Typical metrics | Release velocity, throughput, uptime, adoption, delivery speed |
| Technical signals | APIs, architecture trade-offs, modernization, DevOps collaboration |
| Leadership signals | Balancing engineering, sales, customer, and executive constraints |
| Product OS assets | Logix PLB, Platform Before Intelligence principle |
| Best-fit evidence | Logix, Mahindra Comviva |
| Skills to prioritize | Platform modernization, architecture strategy, AI readiness |
| Risks to avoid | Over-indexing on engineering detail without product outcomes |

## Product Lead / Lead Product Manager

| Dimension | Guidance |
| --- | --- |
| Core hiring problem | Can this person lead ambiguous product areas and align teams? |
| Expected competencies | Strategy, prioritization, stakeholder alignment, team influence |
| Typical metrics | Business outcomes, velocity, adoption, reliability, growth |
| Technical signals | Depends on domain; must show systems thinking |
| Leadership signals | Decision-making under constraints, influence without overclaiming authority |

| Skills to prioritize | Product leadership, roadmap strategy, operating principles |
| Risks to avoid | Inflating people-management scope |
| Interview themes | Leadership judgment, ambiguity, sequencing |
