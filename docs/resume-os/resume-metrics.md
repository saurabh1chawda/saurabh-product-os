# Resume Metrics

## Executive Summary

Resume OS measures whether tailored resumes create qualified interview opportunities.

Metrics should improve decisions without creating false certainty. Resume outcomes are affected by market timing, referrals, role fit, competition, recruiter behavior, and company process. Small samples should be interpreted carefully.

## North Star Metric

Qualified Interview Rate.

Definition:

```text
Qualified interviews / tailored resumes submitted
```

A qualified interview is a recruiter screen, hiring-manager conversation, or interview stage for a role that matches the intended target profile.

## Supporting Metrics

| Metric | Definition | Type |
| --- | --- | --- |
| Resume tailoring time | Time from JD intake to final resume | Leading |
| ATS pass rate | Percentage with no ATS structural blockers | Leading |
| Recruiter response rate | Applications with recruiter response | Lagging |
| Referral conversion | Referrals that lead to recruiter or HM engagement | Lagging |
| Recruiter screen rate | Applications that reach recruiter screen | Lagging |
| Hiring manager interview rate | Applications that reach hiring-manager review/interview | Lagging |
| Interview-to-offer rate | Interviews that convert to offer | Lagging |
| Product OS click-through rate | Resume-driven Product OS visits where measurable | Leading/Lagging |
| Resume variant conversion by archetype | Interview rate by archetype | Lagging |
| Resume component performance | Outcomes by headline, summary, project, or bullet set | Lagging |
| Application volume | Count of submitted tailored applications | Activity |
| Follow-up completion rate | Follow-ups completed after application | Leading |

## Leading Indicators

Leading indicators show whether the process is healthy before outcomes arrive:

- Tailoring time under target
- QA score above threshold
- ATS pass rate
- Link verification pass
- Archetype clarity
- Recruiter scan score
- Referral readiness
- Follow-up completion

## Lagging Indicators

Lagging indicators show market response:

- Recruiter response rate
- Recruiter screen rate
- Hiring manager interview rate
- Interview-to-offer rate
- Offer rate
- Rejection reason patterns

## Component Performance

Track performance by:

- Target headline
- Summary module
- Primary archetype
- Secondary archetypes
- Company-specific project inclusion
- Product OS link selection
- GitHub link inclusion
- Bullet group
- Skills ordering

Use component data to improve selection rules, not to overfit one-off outcomes.

## Measurement Cautions

Do not assume:

- A rejection proves the resume failed.
- A response proves every component worked.
- One role archetype has better market fit from one application.
- Product OS click-through causes interview conversion.
- ATS keyword coverage alone predicts success.

Use metrics as decision support, not false causality.

## Reporting Cadence

| Cadence | Review |
| --- | --- |
| Per application | Resume version, QA score, follow-up status |
| Weekly | Application volume, tailoring time, response status |
| Monthly | Archetype performance, component patterns, conversion rates |
| Quarterly | Resume OS roadmap, canonical evidence updates, role strategy |

## Minimum Dataset Before Strong Conclusions

| Analysis | Minimum Suggested Sample |
| --- | --- |
| Component performance | 10 or more uses |
| Archetype conversion | 10 or more applications per archetype |
| Headline comparison | 5 or more uses per variant |
| Product OS link impact | Meaningful click data plus interview outcome |
| Referral conversion | 5 or more referrals |

## Success Definition

Resume OS is working when:

- Tailoring consistently takes 20 to 30 minutes for standard roles.
- Resumes pass factual and ATS review.
- Recruiters can understand role fit quickly.
- Hiring managers see product judgment.
- Product OS receives qualified traffic from resume links.
- Outcome data improves future resume selection.
