# Competency Taxonomy

## Purpose

The competency taxonomy maps job-description language to stable Resume OS labels. It supports evidence retrieval, ATS coverage, recruiter readability, and interview defensibility.

## Usage Rules

- One JD phrase may map to multiple competencies, but each mapping must be explainable.
- Competency extraction must preserve the original JD phrase.
- Competencies without supporting evidence should appear as gaps, not resume claims.
- Competencies used only in boilerplate receive low weighting.
