## Product OS

Product OS is not a traditional portfolio.

It is a decision-support product for evaluating how I think as a Product Manager before the interview.

It exists because resumes show where someone worked, but they rarely show how someone makes product decisions. Product OS documents the product judgment behind outcomes: customer discovery, validation, prioritization, trade-offs, AI strategy, platform thinking, execution, and measurement.

GitHub complements Product OS by turning that thinking into inspectable artifacts:

- Source code
- Frameworks
- Templates
- Product tools
- Documentation systems
- Representative artifacts

## Operating Principle

> Every repository should leave another Product Manager better than they arrived.

That means every public repository should teach something useful:

- A product decision
- A reusable framework
- A clearer template
- A better way to evaluate trade-offs
- A practical way to connect customer value, technical choices, and business outcomes
