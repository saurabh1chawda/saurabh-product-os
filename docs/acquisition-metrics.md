# Acquisition Metrics

Status: RC1 metrics model  
Last updated: July 2026

## Executive Summary

Acquisition metrics measure whether the Product Leadership Platform is attracting qualified audiences and moving them toward evidence, trust, and interview conversations.

The goal is not traffic volume alone. The goal is qualified attention that turns into recruiter confidence, hiring manager engagement, artifact reuse, or direct conversation.

## North Star Metric

Qualified Interview Opportunities

Definition:

Meaningful hiring or collaboration opportunities from recruiters, hiring managers, founders, product leaders, or executives who engaged with Product OS, GitHub, or derived content.

Examples:

- Recruiter outreach referencing Product OS.
- Hiring manager asking about a Product Leadership Brief.
- Interview request after Executive Brief view.
- Founder or product leader asking for a conversation.
- Referral that cites Product OS or GitHub artifacts.

## Acquisition Funnel

```text
Qualified discovery
  |
Executive context
  |
Evidence engagement
  |
Artifact credibility
  |
Contact / interview intent
```

## Supporting Metrics

| Metric | Definition | Primary Signal |
| --- | --- | --- |
| Qualified discovery visits | Visitors from LinkedIn, GitHub, referrals, search, or talks | Top-of-funnel reach |
| Executive Brief view rate | Share of visitors reaching `/executive` | Fast context adoption |
| Evidence engagement rate | Visits to Product Leadership Briefs, Playbook, or Operating Principles | Trust-building depth |
| GitHub artifact engagement | Profile visits, repo visits, stars, forks, README clicks | Reusable artifact credibility |
| Contact conversion rate | Email, LinkedIn, or Contact clicks | Interview intent |
| Return visitor rate | Visitors returning after first session | Sustained trust |
| Content reuse rate | Template use, forks, downloads, workshop requests | Community value |
| Role-fit path completion | Executive Brief -> role-fit brief -> Contact | Recruiter conversion quality |

## Channel Metrics

| Channel | Acquisition Metrics |
| --- | --- |
| LinkedIn | Profile visits, Product OS clicks, qualified DMs |
| GitHub | Profile visits, repository visits, stars, Product OS return clicks |
| Product OS | Executive Brief views, brief engagement, contact conversions |
| Newsletter | Opens, clicks, replies, returning visitors |
| Blog | Organic visits, source clicks, time on page |
| Workshops | Attendance, artifact reuse, follow-up requests |
| Podcasts | Referral visits, inbound conversations |
| Conference talks | Product OS visits, invitations, follow-up conversations |

## Measurement Guidance

Review weekly during active launch periods:

- Which channel drove qualified discovery?
- Which asset converted attention into evidence engagement?
- Which pages created contact intent?
- Which audiences returned?

Review monthly:

- Which audience journey is strongest?
- Which flagship artifact is under-distributed?
- Which content types create the most qualified traffic?
- Which CTAs need refinement?

## Early Benchmarks

Initial targets should be directional until traffic volume stabilizes.

| Metric | Initial Target |
| --- | --- |
| Homepage to Executive Brief click-through | 25%+ of recruiter-intent visitors |
| Executive Brief to evidence click-through | 15%+ |
| Executive Brief to Contact click-through | 8-12% |
| GitHub to Product OS return clicks | 5-10% |
| Product Leadership Brief to Contact click-through | 5-8% |
| Returning visitor rate | 10%+ |

## Quality Signals

Track qualitative signals:

- Recruiters reference the Executive Brief.
- Hiring managers cite a specific Product Leadership Brief.
- Engineering leaders mention GitHub artifact quality.
- Product managers reuse templates.
- Interview questions become deeper because basic context is already clear.

## Anti-Patterns

- Optimizing for broad reach without role-fit engagement.
- Treating GitHub stars as a hiring proxy.
- Counting traffic that never reaches Product OS.
- Measuring content output volume as acquisition success.
- Ignoring qualitative hiring signals.

## Operating Principle

Acquisition is working when the right people reach the right evidence faster.
