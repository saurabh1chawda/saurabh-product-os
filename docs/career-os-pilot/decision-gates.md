# Career OS Pilot Decision Gates

## Purpose

Decision gates ensure that every application moves through the Career OS Pilot Workflow with evidence, approval, and traceability.

## Gate 1 — Proceed / Reject Opportunity

| Field | Definition |
| --- | --- |
| Decision | Decide whether a role should enter the pilot workflow. |
| Evidence | Opportunity intake, source URL, role title, company, location/work mode, duplicate check. |
| Required Approver | Saurabh Chawda. |
| Possible Outcomes | Proceed, reject, park for more information. |

## Gate 2 — Proceed with Resume Generation

| Field | Definition |
| --- | --- |
| Decision | Decide whether JD Intelligence supports resume planning and generation. |
| Evidence | JD Snapshot, JD Intelligence report, archetype, seniority, mandatory requirements, evidence gaps. |
| Required Approver | Saurabh Chawda. |
| Possible Outcomes | Proceed, revise qualification, reject, human review required. |

## Gate 3 — QA Approval

| Field | Definition |
| --- | --- |
| Decision | Decide whether the resume is safe and strong enough for submission. |
| Evidence | Resume draft, evidence map, QA checklist, ATS review, factual review, recruiter first-impression review. |
| Required Approver | Saurabh Chawda. |
| Possible Outcomes | Approve, revise resume, revise strategy, reject application. |

## Gate 4 — Ready for Submission

| Field | Definition |
| --- | --- |
| Decision | Decide whether the application can be manually submitted. |
| Evidence | Approved resume export, JD Snapshot, application source/channel, required portal fields. |
| Required Approver | Saurabh Chawda. |
| Possible Outcomes | Submit, defer, cancel, return to QA. |

## Gate 5 — Interview Complete

| Field | Definition |
| --- | --- |
| Decision | Decide whether interview tracking is complete and next action is clear. |
| Evidence | Interview stage, date, format, preparation task, completion event, privacy-safe review notes. |
| Required Approver | Saurabh Chawda. |
| Possible Outcomes | Continue, follow up, wait, withdraw, record explicit outcome. |

## Gate 6 — Pilot Learning Captured

| Field | Definition |
| --- | --- |
| Decision | Decide whether the application produced enough product learning for reporting. |
| Evidence | Pilot observation, timing, friction, manual overrides, outcome status, validation status. |
| Required Approver | Saurabh Chawda. |
| Possible Outcomes | Complete, request more observation, create decision-log entry, create postmortem, defer learning until outcome matures. |
