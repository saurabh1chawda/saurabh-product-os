# JD Snapshot Template

## Purpose

Record a stable local reference for the job description so future analysis does not depend on a live posting remaining available.

## Required Fields

- JD Snapshot ID: `[JD-YYYY-###]`
- Opportunity ID: `[OPP-YYYY-###]`
- Capture date: `[YYYY-MM-DD]`
- Company: `[Company name]`
- Role title: `[Role title]`
- Source URL: `[URL]`
- Posting ID: `[Posting ID or unknown]`
- Snapshot file path: `[Local path]`
- Capture method: `[Manual copy / PDF / screenshot / saved page]`
- JD version note: `[Initial / updated / replaced]`

## Optional Fields

- Application deadline: `[YYYY-MM-DD or unknown]`
- Compensation range: `[Private; store only if needed in private workspace]`
- Recruiter source: `[Private; store only if needed]`

## Required Evidence

- Stable JD snapshot.
- Source URL.
- Capture date.

## Completion Checklist

- [ ] JD snapshot is saved in the correct private or operational location.
- [ ] Live URL is not the only source of truth.
- [ ] No unnecessary private data is included.
- [ ] JD Snapshot ID is ready for JD Intelligence.

## Example Placeholders

- JD Snapshot ID: `JD-2026-001`
- Snapshot file path: `[docs/career-os-pilot/pilot-workspace/artifacts/... ]`
