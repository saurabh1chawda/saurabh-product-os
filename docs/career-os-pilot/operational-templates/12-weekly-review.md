# Weekly Review Template

## Purpose

Review active pilot operations, validation status, privacy posture, friction, and directional metrics.

## Required Fields

- Review ID: `[WEEKLY-YYYY-###]`
- Review week: `[YYYY-W##]`
- Review date: `[YYYY-MM-DD]`
- Applications reviewed: `[Count]`
- Open applications: `[Count]`
- Closed applications: `[Count]`
- Validation status: `[Pass / warnings / fail]`
- Privacy validation status: `[Pass / warnings / fail]`
- Top operational issues: `[List]`
- Decisions needed: `[List]`
- Next-week priorities: `[List]`

## Optional Fields

- Directional funnel notes: `[Notes with sample sizes]`
- Console usefulness notes: `[Notes]`
- Candidate workflow confidence: `[1-5]`

## Required Evidence

- Registry validation.
- Privacy validation.
- Pilot observations.
- Defect register references.

## Completion Checklist

- [ ] Metrics include denominators.
- [ ] Immature outcomes are not treated as terminal.
- [ ] Privacy posture is checked.
- [ ] Defects and decisions are routed appropriately.

## Example Placeholders

- Review week: `[2026-W30]`
- Validation status: `[Pass]`
