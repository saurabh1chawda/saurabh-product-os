# Final Pilot Report Template

## Purpose

Summarize pilot outcomes, directional evidence, operational learning, and roadmap recommendations.

## Required Fields

- Report ID: `[PILOT-REPORT-YYYY-###]`
- Reporting date: `[YYYY-MM-DD]`
- Pilot period: `[Start date - end date]`
- Applications submitted: `[Count]`
- Mature outcomes: `[Count]`
- Immature applications: `[Count]`
- Validation summary: `[Pass / warnings / fail]`
- Privacy summary: `[Pass / warnings / fail]`
- Workflow success summary: `[Summary]`
- Directional funnel findings: `[Findings with denominators]`
- Resume and positioning findings: `[Findings with sample sizes]`
- Operational friction themes: `[List]`
- Defects summary: `[P0 / P1 / P2 / P3 counts]`
- Decisions made: `[Decision IDs]`
- Roadmap recommendations: `[List]`
- Final pilot decision: `[GO / REVISE / STOP]`

## Optional Fields

- Representative anonymized examples: `[Examples]`
- Deferred questions: `[List]`

## Required Evidence

- Application Registry metrics.
- Privacy validation.
- Weekly reviews.
- Pilot observations.
- Decision log.

## Completion Checklist

- [ ] Results are directional when sample size is small.
- [ ] No private application data is included.
- [ ] Metrics include denominators.
- [ ] Roadmap recommendations are evidence-based.
- [ ] Final decision is explicit.

## Example Placeholders

- Applications submitted: `[15]`
- Final pilot decision: `[GO]`
