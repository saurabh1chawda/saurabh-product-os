# Resume Strategy Template

## Purpose

Define the approved resume positioning before assembly. This template supports Gate 2 in [Decision Gates](../decision-gates.md).

## Required Fields

- Resume Strategy ID: `[STRAT-YYYY-###]`
- JD Intelligence ID: `[JDI-YYYY-###]`
- Target headline: `[Headline]`
- Summary direction: `[Summary approach]`
- Primary archetype: `[Archetype]`
- Secondary archetypes: `[Archetype list]`
- Selected achievement IDs: `[Achievement IDs]`
- Selected bullet libraries: `[Library list]`
- Selected project or Product OS asset: `[One primary selection]`
- Skills priority: `[Ordered skill groups]`
- Known gaps and mitigations: `[List]`
- Gate 2 decision: `[Approve / revise / reject]`

## Optional Fields

- Alternative positioning: `[Notes]`
- Role-specific risks: `[Notes]`
- Manual override candidates: `[List]`

## Required Evidence

- JD Intelligence report.
- Canonical evidence mappings.
- Product OS asset mapping.

## Completion Checklist

- [ ] Strategy uses verified evidence only.
- [ ] One primary positioning story is clear.
- [ ] Product OS is used as proof, not filler.
- [ ] Human approver is recorded.

## Example Placeholders

- Target headline: `[Senior Product Manager - Monetization and Growth]`
- Gate 2 decision: `[Approve]`
