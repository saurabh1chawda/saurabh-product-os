# Postmortem Template

## Purpose

Document significant pilot defects, privacy incidents, workflow failures, or learning events after resolution or containment.

## Required Fields

- Postmortem ID: `[PM-YYYY-###]`
- Date opened: `[YYYY-MM-DD]`
- Date closed: `[YYYY-MM-DD or open]`
- Severity: `[P0 / P1 / P2 / P3]`
- Summary: `[Privacy-safe summary]`
- Trigger: `[What happened]`
- Impact: `[Applications or workflow affected]`
- Root cause: `[Known / unknown / under review]`
- Detection method: `[How found]`
- Resolution: `[What changed or what process changed]`
- Preventive action: `[Action]`
- Owner: `[Owner]`

## Optional Fields

- Timeline: `[Privacy-safe timeline]`
- Related defects: `[IDs]`
- Related decisions: `[Decision IDs]`

## Required Evidence

- Defect report or incident record.
- Validation result after fix or containment.
- Product Owner review.

## Completion Checklist

- [ ] Private data is excluded.
- [ ] Root cause is not invented.
- [ ] Follow-up owner is clear.
- [ ] Preventive action is recorded.

## Example Placeholders

- Severity: `[P1]`
- Detection method: `[Weekly review]`
