# Decision Log Entry Template

## Purpose

Record why a significant pilot product decision was made. This template complements [pilot-decision-log.md](../pilot-decision-log.md).

## Required Fields

- Decision ID: `[DEC-YYYY-###]`
- Date: `[YYYY-MM-DD]`
- Pilot phase: `[Phase]`
- Problem statement: `[Problem]`
- Observed evidence: `[Evidence]`
- Supporting metrics: `[Metrics with denominators]`
- Alternatives considered: `[List]`
- Decision: `[Decision]`
- Reasoning: `[Why this decision]`
- Expected impact: `[Impact]`
- Risks: `[Risks]`
- Validation plan: `[Plan]`
- Review date: `[YYYY-MM-DD]`
- Status: `[Proposed / approved / implemented / deferred / rejected / superseded]`

## Optional Fields

- Related defect IDs: `[IDs]`
- Related application references: `[Anonymized references]`

## Required Evidence

- Pilot observation, metric, validation result, or defect evidence.
- Product Owner approval where required.

## Completion Checklist

- [ ] Decision explains why, not just what.
- [ ] Evidence is privacy-safe.
- [ ] Status is valid.
- [ ] Review date is recorded.

## Example Placeholders

- Decision ID: `DEC-2026-001`
- Status: `[Proposed]`
