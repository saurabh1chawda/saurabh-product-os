# Pilot Observation Template

## Purpose

Capture product learning from each application workflow so pilot findings can guide future roadmap decisions.

## Required Fields

- Observation ID: `[OBS-YYYY-###]`
- Application ID: `[APP-YYYY-###]`
- Observation date: `[YYYY-MM-DD]`
- Workflow stage observed: `[Stage]`
- Time spent: `[Minutes]`
- Friction observed: `[Description]`
- Severity: `[P0 / P1 / P2 / P3 / none]`
- Workaround used: `[Yes / No; summary]`
- Reusable learning: `[Yes / No]`
- Suggested improvement: `[Evidence-based suggestion]`

## Optional Fields

- Command or workflow repetition: `[Notes]`
- Missing information: `[Notes]`
- Unused information: `[Notes]`
- Confidence rating: `[1-5]`

## Required Evidence

- Stage artifact reference.
- Timing estimate.
- Specific friction or success signal.

## Completion Checklist

- [ ] Observation is evidence-based.
- [ ] Private data is excluded.
- [ ] Hypotheses are labeled.
- [ ] Decision-log entry is created if needed.

## Example Placeholders

- Workflow stage observed: `[QA Review]`
- Friction observed: `[Manual override category unclear]`
