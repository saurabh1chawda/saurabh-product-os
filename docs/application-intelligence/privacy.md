# Application Intelligence Privacy

Last updated: 2026-07-17

## Do Not Store

- Passwords.
- Recruiter private notes.
- Sensitive personal data unrelated to hiring outcomes.
- Confidential interview questions.
- Employer confidential information.
- Anything unrelated to application, resume, or funnel measurement.

## Storage Principles

- Store references to resumes, not resume content.
- Store job-description hashes and paths, not unnecessary copyrighted copies.
- Store compensation only when volunteered in the hiring process.
- Store rejection reasons only when provided or manually classified by the candidate.
- Keep analytics local-first until a deliberate sync strategy exists.

## Access

Application Intelligence should be private by default. Public portfolio artifacts should never expose application-level outcomes or identifiable recruiter interactions.
