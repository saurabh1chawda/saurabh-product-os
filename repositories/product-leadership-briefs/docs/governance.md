# Governance

Status: Draft  
Last updated: July 2026

## Versioning

Product Leadership Briefs should use simple semantic versions for public milestones:

- `v1.x` for repository foundation, framework, template, and brief navigation.
- `v2.x` for evidence packs, impact dashboard patterns, and facilitation notes.
- `v3.x` for domain-specific brief variants and reusable creation kits.

## Documentation Ownership

Documentation is the product. Every brief, framework, and template should help visitors evaluate or practice product judgment.

Repository owners should keep:

- README current.
- Decision index accurate.
- Product OS links working.
- Brief statuses visible.
- Template aligned to the Product Decision Framework.
- Roadmap aligned to actual maturity.

## Launch Readiness

Before public launch, confirm:

- README explains the audience, problem, and value.
- Decision framework is reusable.
- Leadership Brief template is complete.
- Brief library navigation is clear.
- Examples do not duplicate Product OS narratives.
- License decision is complete.
- Product OS links are valid.

## Repository Principles

- Document decisions, not just outcomes.
- Make trade-offs explicit.
- Treat stakeholder disagreement with fairness.
- Use metrics as evidence, not decoration.
- Extract principles that travel beyond the original context.
- Keep the format useful for other Product Managers.
