# Roadmap

Status: v1 foundation  
Last updated: July 2026

## Version 1: Core Frameworks

Goal: Establish the repository as a coherent AI product operating guide.

Scope:

- README
- Framework index
- Template index
- Example structure
- Architecture document
- Governance document
- Product specification
- Roadmap

Exit criteria:

- A visitor can understand what the playbook is, who it is for, and how to navigate it.
- Frameworks have clear placeholder pages and a consistent future structure.
- Templates and examples have clear launch direction.

## Version 2: Templates

Goal: Turn the repository into a reusable product toolkit.

Scope:

- AI PRD
- AI Experiment Canvas
- AI Opportunity Assessment
- AI Success Metrics Canvas
- Prompt Review Checklist
- Worked examples
- Workshop facilitation notes

Exit criteria:

- A Product Manager can copy at least three templates into real product work.
- Examples show how the templates are applied.
- Each template links to a related framework.

## Version 3: Interactive Tools

Goal: Make the playbook easier to apply through lightweight interactive artifacts.

Scope:

- Scoring worksheets
- AI opportunity review flow
- Automation depth review
- Trust readiness checklist
- Metrics mapping worksheet

Exit criteria:

- A team can run an AI product review using the playbook without needing Product OS.
- Interactive tools remain lightweight, transparent, and easy to inspect.
