# Governance

Status: Draft  
Last updated: July 2026

## Versioning

AI Product Playbook should use simple semantic versions for public milestones:

- `v1.x` for repository foundation and core framework structure.
- `v2.x` for reusable templates and examples.
- `v3.x` for interactive tools and portable review kits.

## Documentation Ownership

Documentation is the product. Every framework, template, and example should be maintained as a usable artifact, not a placeholder page.

Repository owners should keep:

- README current.
- Framework index accurate.
- Template status visible.
- Product OS links working.
- Roadmap aligned to actual repository maturity.

## Launch Readiness

Before public launch, confirm:

- README explains the audience, problem, and value.
- Framework index is coherent.
- Template index is coherent.
- Examples do not duplicate Product Leadership Briefs.
- License decision is complete.
- Product OS links are valid.
- Roadmap reflects the current state.

## Repository Principles

- Start with customer workflow, not model capability.
- Build evidence before confidence.
- Design trust before automation.
- Measure customer outcomes before model metrics.
- Keep artifacts practical enough for real product work.
- Avoid AI hype and vendor promotion.
