# Framework Index

Status: v1 navigation layer  
Last updated: July 2026

This index organizes the planned AI Product Playbook frameworks. The goal of v1 is to define coherent navigation and scope before fully implementing each framework.

| Framework | Purpose | Status |
| --- | --- | --- |
| [AI Prioritization](ai-prioritization/README.md) | Decide which AI opportunity to build first and why | Planned |
| [Build vs Buy AI](build-vs-buy-ai/README.md) | Decide whether to buy, configure, extend, build, or platformize an AI capability | Planned |
| [RAG vs Agent](rag-vs-agent/README.md) | Choose the simplest architecture that reliably solves the customer problem | Planned |
| [AI Success Metrics](ai-success-metrics/README.md) | Measure customer outcomes before model performance | Planned |
| [Trust Before Automation](trust-before-automation/README.md) | Evaluate trust safeguards before increasing automation depth | Planned |

| [Opportunity Scoring](opportunity-scoring/README.md) | Assess AI opportunity quality using customer, business, data, trust, and cost signals | Planned |

## Framework Design Standard

Each framework should eventually include:

- Problem it solves
- When to use it
- Inputs required
- Decision questions
- Scorecard or decision model
- Example application
- Common mistakes
- Stop criteria
- Product OS reference link
