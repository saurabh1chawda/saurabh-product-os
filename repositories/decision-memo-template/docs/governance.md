# Governance

Status: Draft  
Last updated: July 2026

## Versioning

Decision Memo Template should use simple semantic versions for public milestones:

- `v1.x` for repository foundation, frameworks, templates, and example placeholders.
- `v2.x` for sample memos, decision review checklist, and decision log artifacts.
- `v3.x` for portable formats and decision quality scoring.

## Documentation Ownership

Documentation is the product. Every artifact should help Product Managers communicate decisions with more clarity and evidence.

Repository owners should keep:

- README current.
- Templates easy to copy.
- Frameworks practical.
- Examples concise and non-duplicative.
- Product OS links working.
- Roadmap aligned to repository maturity.

## Launch Readiness

Before public launch, confirm:

- README explains the audience, problem, and value.
- Templates are reusable.
- Frameworks support executive communication.
- Decision Writing Pyramid is clear.
- Common Memo Mistakes are visible.
- License decision is complete.
- Product OS links are valid.

## Repository Principles

- Lead with the recommendation.
- Support recommendations with evidence.
- Make trade-offs explicit.
- Name alternatives and assumptions.
- Keep background proportional.
- Make the decision request visible.
