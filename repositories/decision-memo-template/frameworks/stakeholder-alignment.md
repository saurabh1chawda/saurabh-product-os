# Stakeholder Alignment

Status: v1 framework  
Last updated: July 2026

## Purpose

Stakeholder alignment in a decision memo should explain how competing priorities were understood and resolved.

Alignment does not mean everyone wanted the same thing. It means the decision makes the trade-off explicit enough that teams can move forward.

## Stakeholder Map

| Stakeholder | Optimized For | Concern | Alignment Move |
| --- | --- | --- | --- |
| Product | Customer and business outcome |  |  |
| Engineering | Feasibility, reliability, maintainability |  |  |
| Design | Usability and workflow quality |  |  |
| Data | Measurement and evidence quality |  |  |
| Sales / GTM | Customer urgency and revenue |  |  |
| Leadership | Strategy, cost, and timing |  |  |

## Alignment Questions

- Who disagrees with the recommendation?
- What are they optimizing for?
- Which concern is valid?
- Which concern changes the decision?
- Which concern becomes a risk to monitor?
- What decision rule resolves the disagreement?

## Writing Guidance

Do not flatten disagreement into false consensus.

Strong memos make disagreement useful by showing how each function's constraint shaped the final recommendation.
