# Governance

Status: Draft  
Last updated: July 2026

## Versioning

Product Discovery Toolkit should use simple semantic versions for public milestones:

- `v1.x` for repository foundation, frameworks, templates, examples, and anti-patterns.
- `v2.x` for workshop facilitation, industry variants, and evidence scoring improvements.
- `v3.x` for a complete discovery operating kit and editable formats.

## Documentation Ownership

Documentation is the product. Every artifact should help Product Managers gather better evidence before making decisions.

Repository owners should keep:

- README current.
- Frameworks practical.
- Templates easy to copy.
- Examples concise and non-duplicative.
- Product OS links working.
- Roadmap aligned to repository maturity.

## Launch Readiness

Before public launch, confirm:

- README explains the audience, problem, and value.
- Toolkit is independently useful.
- Evidence Ladder is clear.
- Discovery Anti-Patterns are visible.
- Templates support real discovery work.
- License decision is complete.
- Product OS links are valid.

## Repository Principles

- Validate before building.
- Separate requests from needs.
- Treat evidence quality as a product decision input.
- Use interviews to learn, not confirm.
- Prefer observed behavior over stated intent.
- Keep artifacts lightweight enough for real teams to use.
