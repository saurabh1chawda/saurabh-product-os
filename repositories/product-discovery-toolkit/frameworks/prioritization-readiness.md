# Prioritization Readiness

Status: v1 framework  
Last updated: July 2026

## Purpose

Prioritization Readiness helps teams decide whether an opportunity has enough evidence to enter roadmap prioritization.

## Readiness Checklist

| Question | Yes / No | Evidence |
| --- | --- | --- |
| Is the customer segment clear? |  |  |
| Is the workflow understood? |  |  |
| Is the problem repeated across customers or behavior? |  |  |
| Is the cost of the problem visible? |  |  |
| Are key assumptions documented? |  |  |
| Has the riskiest assumption been tested? |  |  |
| Is business impact measurable? |  |  |
| Are success criteria defined? |  |  |
| Are dependencies known? |  |  |
| Would negative evidence change the roadmap? |  |  |

## Readiness Levels

### Ready

The opportunity has clear customer evidence, measurable business value, known assumptions, and defined success criteria.

### Needs Validation

The opportunity appears meaningful, but one or more assumptions must be tested before prioritization.

### Needs Reframing

The team has signal, but the problem statement is too broad, too solution-led, or too stakeholder-driven.

### Not Ready

The opportunity depends mainly on opinion, pressure, or weak evidence.

## Recommendation

Do not prioritize an opportunity until the team can explain:

- Who has the problem.
- What workflow is affected.
- Why it matters.
- What evidence supports it.
- What decision will change if validation succeeds or fails.
