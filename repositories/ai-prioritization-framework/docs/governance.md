# Governance

Status: Draft  
Last updated: July 2026

## Versioning

AI Prioritization Framework should use simple semantic versions for public milestones:

- `v1.x` for repository foundation, framework, templates, examples, and anti-patterns.
- `v2.x` for downloadable templates, facilitation guidance, and more examples.
- `v3.x` for interactive tools and portfolio-level prioritization support.

## Documentation Ownership

Documentation is the product. Every framework, template, and example should improve AI roadmap decision quality.

Repository owners should keep:

- README current.
- Framework documents consistent.
- Templates easy to copy.
- Examples concise and non-duplicative.
- Product OS links working.
- Roadmap aligned to repository maturity.

## Launch Readiness

Before public launch, confirm:

- README explains the audience, problem, and value.
- Framework is independently usable.
- Scoring model and criteria are clear.
- Anti-patterns are visible.
- Templates can support a real prioritization discussion.
- License decision is complete.
- Product OS links are valid.

## Repository Principles

- Prioritize customer value over AI novelty.
- Use scoring to improve judgment, not replace it.
- Make anti-patterns visible.
- Prefer simpler software when it solves the problem.
- Treat trust and readiness as first-class prioritization criteria.
- Keep the framework practical enough for roadmap planning.
